# WoW.Two.Sdk.Backend.Beta.Media.Captions

> Pure WebVTT caption parsing — no network, no external dependencies. Parses YouTube / WebVTT caption text into ordered, cleaned segments.

Part of the `WoW2.Sdk.Backend.Beta` mono-lib (folder `src/Media/Captions/`).

## Usage

```csharp
using WoW.Two.Sdk.Backend.Beta.Media.Captions;

builder.Services.AddVttCaptionParser();

public sealed class MyService(IVttCaptionParser captions)
{
    public IReadOnlyList<CaptionSegment> Read(string vtt) => captions.Parse(vtt);
}
```

## Types

| Type | Role |
|---|---|
| `IVttCaptionParser` / `VttCaptionParser` | Parse WebVTT → `IReadOnlyList<CaptionSegment>` (singleton, stateless) |
| `CaptionSegment` | One timed line — `Start`, `End`, `Text` |
| `CaptionTrack` | A language's caption track — `Language`, `IsAutoGenerated`, `Segments` |

## Behavior

- Strips inline per-word timing + styling tags (`<00:00:00.320><c>…</c>`).
- Decodes HTML entities (`&amp;` → `&`).
- De-duplicates rolling auto-caption overlays (identical text, adjacent timing).
- Tolerates cue identifiers, timing-line settings suffixes, comma **or** dot decimals, and CRLF / LF endings.
- Blank input or a cue-less header returns an empty list — never throws on malformed cues (they're skipped).

## Scope

**VTT only.** SRT / TTML / JSON3 and a multi-format `ICaptionParser` facade land when a second format is actually needed (extracted from a consuming product, not built speculatively).

## Provenance

Extracted from the TranscriptForge POC (`10x-ventures-transcript-forge`) — the first proven consumer — as the pure, zero-dependency leaf of the media/transcript family. STT provider, audio extraction, and the transcript orchestrator remain product-inline until a second impl or consumer justifies the seams.
