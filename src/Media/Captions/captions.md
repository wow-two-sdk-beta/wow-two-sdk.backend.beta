# WoW.Two.Sdk.Backend.Beta.Media.Captions

> Pure caption parsing across four formats — WebVTT, SubRip, TTML/DFXP, YouTube json3 — into ordered, cleaned segments. No network, no external dependencies.

Part of the `WoW2.Sdk.Backend.Beta` mono-lib (folder `src/Media/Captions/`).

## Usage

```csharp
using WoW.Two.Sdk.Backend.Beta.Media.Captions;

builder.Services.AddCaptionParsing();   // all parsers + facade + writers

public sealed class MyService(ICaptionParser captions)
{
    // Format known:
    IReadOnlyList<CaptionSegment> a = captions.Parse(srt, CaptionFormat.SubRip);
    // Format unknown — auto-detect:
    IReadOnlyList<CaptionSegment> b = captions.Parse(unknownText);
    // As a language-tagged track:
    CaptionTrack track = captions.ParseTrack(vtt, "en", isAutoGenerated: true);
}
```

Need only one format? `AddVttCaptionParser()` still registers just the VTT parser.

## Types

| Type | Role |
|---|---|
| `ICaptionParser` / `CompositeCaptionParser` | Format-aware facade — detect + dispatch; `Parse` / `TryParse` / `ParseTrack` |
| `IVttCaptionParser` · `ISrtCaptionParser` · `ITtmlCaptionParser` · `IJson3CaptionParser` | Per-format parsers (singleton, stateless) |
| `CaptionFormat` | `WebVtt` · `SubRip` · `Ttml` · `YouTubeJson3` |
| `CaptionFormatDetector` | `TryDetect(content, out format)` — sniffs the format from leading content |
| `ICaptionWriter` / `VttCaptionWriter` · `SrtCaptionWriter` | Serialize segments back out — parse-then-write across formats = conversion |
| `CaptionSegment` | One timed line — `Start`, `End`, `Text` |
| `CaptionTrack` | A language's track — `Language`, `IsAutoGenerated`, `Segments` |

## Behavior

- Every parser strips inline markup, decodes HTML entities, collapses whitespace, and **never throws** on malformed input — bad cues are skipped, blank/unparseable input returns an empty list.
- **VTT:** strips per-word timing tags; de-dupes YouTube rolling overlays; tolerates cue ids, settings suffixes, comma/dot decimals, CRLF/LF.
- **SRT:** blank-line-separated cues; tolerant of missing/out-of-order index lines.
- **TTML/DFXP:** namespace-agnostic `<p>` cues; `begin`+`end` or `begin`+`dur`; `<br/>` → space.
- **json3:** one segment per text-bearing event; skips window/position-only events.
- **Detection:** `WEBVTT` header → VTT · `{ "events" }` → json3 · `<tt>`/ttml → TTML · comma `-->` → SRT · dot `-->` → VTT.

## Conversion

```csharp
var segs = captions.Parse(json3, CaptionFormat.YouTubeJson3);
string srt = new SrtCaptionWriter().Write(segs);   // json3 → SubRip
```

## Provenance

The VTT leaf was extracted from the TranscriptForge POC (`10x-ventures-transcript-forge`). SRT/TTML/json3 + the `ICaptionParser` facade + writers complete the caption *vector* (own logic, zero-dep) so the next transcript/subtitle consumer finds it whole. STT provider, audio extraction, and the transcript orchestrator remain product-inline until a second impl justifies the seams.
